The Gateway

Seven AI providers, one front door. Every call gets routed, rate-limited, retried — and written into a ledger.

An air traffic control tower

The agent never calls an AI provider directly. Every request flies through the Gateway (gateway/main.py) — a control tower with seven runways: Gemini, Groq, Cerebras, NVIDIA, OpenRouter, GitHub Models, and a local Ollama.

The tower’s job, on every single call:

🛣️
Assign a runway

Pick a provider that has the needed capabilities (tools, vision, structured output) and free capacity right now.

⏳
Respect the congestion rules

Each provider has a rate limit. The tower tracks calls per minute, per day, and tokens spent, and never sends a plane to a full runway.

🔁
Reroute on trouble

Provider down or busy? The call falls through a failover ladder to the next one. The caller never notices.

📑
Log every landing

Every call — tokens, latency, which skill made it, which session — lands in a SQLite ledger. Cost accounting comes free.

Who flies on which airline

Each skill is pinned to a preferred provider in one tiny file — matching the job to the model’s strengths:

CODE — gateway/agent_routing.yaml 

planner: gemini
researcher: gemini
distiller: gemini
summariser: gemini
critic: groq
formatter: gemini
retriever: github
sandbox_executor: github
coder: gemini
browser: gemini
PLAIN ENGLISH

The Planner needs the strongest reasoning — give it Gemini.

The bulky reading-and-writing skills also go to Gemini, which handles very long documents comfortably.

The Critic only says pass or fail — send it to Groq, a small, fast, cheap worker.

Classifier-ish skills ride the free GitHub Models tier.

Changing any of this is a one-line edit. No code.

📌
A pin is a preference, not a promise

If a pinned provider is cooling down, the gateway waits briefly, then falls back through the normal ladder. The system degrades gracefully instead of refusing to fly.

How a runway says “not now”

Before any call, the router asks the provider’s bookkeeping: can I use you right now? Three of its checks:

CODE — gateway/router.py 

wait = limits["cooldown"] - (now - self.last_call)
if wait > 0:
    return False, f"cooldown ({wait:.1f}s)"
if len(self.calls_minute) >= limits["rpm"]:
    return False, "RPM limit"
if self.calls_today >= limits["rpd"]:
    return False, "RPD limit"
PLAIN ENGLISH

Has enough time passed since this provider’s last call? Some need a short breather between requests.

If not — refuse, and say exactly how long the breather has left.

Already at the requests-per-minute ceiling? Refuse.

Already at the requests-per-day ceiling? Refuse.

Every refusal names its reason — so the router can report exactly why it skipped a provider.

Beyond chat, the same front door serves the rest of the system:

/v1/chat The main channel — every skill’s text and tool-use calls
/v1/vision Screenshot questions — what the Browser’s layer 3 calls
/v1/embed Turns text into vectors for the agent’s memory search
/v1/cost/by_agent The ledger, sliced: what each skill spent, per session
