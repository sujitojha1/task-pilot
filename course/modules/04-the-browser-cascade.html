The Browser Cascade

Reading a web page can cost almost nothing — or a vision AI looking at screenshots. The skill always tries the cheap way first.

Getting answers out of a building

Suppose you need information from an office building. You’d read the sign outside first. If that’s not enough, follow the posted directions. Still stuck? Ask at the front desk, where the receptionist reads the building directory to you. Only as a last resort do you walk the halls yourself, looking around room by room.

The Browser skill (code/browser/) treats every web page the same way. Many modern pages are JavaScript-rendered or hide their answers behind filters and buttons, so the Researcher’s simple page-fetch comes back empty. This skill escalates through four layers, stopping at the first one that succeeds:

📋

1 · Extract

Download the page and pull out the readable text. No browser, no AI — nearly free.

🎲

2a · Deterministic

If the caller supplied exact click/fill instructions, replay them in a real browser. No guessing allowed.

🗣️

2b · A11y

A real browser reads the page’s accessibility tree and a text-only AI decides what to click, step by step.

👀

3 · Vision

Screenshot the page, draw numbered tags on every clickable element, and let a vision model literally look.

This is an escalation ladder — a pattern worth requesting by name when you steer an AI assistant: “try the cheap path first and escalate only on failure.” Most pages never get past layer 1, so the average cost stays tiny.

The rule that triggers escalation

How does layer 1 know its answer is not good enough? Two checks — too little text, or a goal that demands interaction:

CODE — code/browser/skill.py 

if len(content) < 200:
    return False
interactive_verbs = ("click", "fill", "select", "type", "drag",
                     "filter", "sort", "submit", "navigate")
if any(v in goal.lower() for v in interactive_verbs):
    return False
return True

🚧
Blocked is a verdict, not a defeat

If any layer finds a CAPTCHA or a login wall, the skill does NOT keep climbing — clicking harder at a wall just wastes money. It returns immediately with the code gateway_blocked, and the Planner is recalled to route around the blocked site. Escalation fixes “too hard”; it cannot fix “not allowed.”
