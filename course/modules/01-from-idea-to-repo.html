From Idea to Repo

You hand it one sentence. It hands you back a GitHub project with a repo, formal requirements, issues, and milestones — with receipts for everything.

One sentence in, a whole project out

repo-genesis is an agent you run from the command line. You give it a raw idea, and it bootstraps a complete GitHub repository around it: researched, specified, and broken into tracked work items.

Think of it as a city planning office. You walk in with a rough sketch of a building. The office checks your permit to build (are you logged in?), surveys comparable buildings nearby, drafts formal blueprints, and issues numbered permits for each piece of work — and every permit cites the blueprint line that justifies it.

The eight steps of a run

Every run walks the same arc — from checking your identity to publishing the plan and replaying what happened:

1
Verify

Confirm you are logged into GitHub. If not, stop cleanly — never act half-authenticated.

2
Bootstrap

Create the repository with a license and a README holding the initial idea.

3
Research

Search GitHub for 3–5 similar projects and read each one's README on the live page.

4
Compare & refine

Build a comparison table of those projects and sharpen the idea against the evidence.

5
Specify

Write formal requirements, each with a stable ID like REQ-001 that can be cited later.

6
Publish

Open one GitHub issue per piece of work, each traced to its requirement, grouped into milestones.

7
Review

A QA critic checks the chain: every requirement has an issue, every issue has a milestone.

8
Replay

Emit a step-by-step record of everything it did, every page it visited, and what it cost.

💡
Traceability is the whole point

Every issue cites a requirement; every requirement cites the refined idea; the refined idea cites real projects it was compared against. Engineers call this a traceability chain — it means no work item exists without a reason on record.

Two machines under the hood

The codebase splits into two independent programs that talk over the network:

🧠
The agent (code/)

Plans the work, runs the skills, drives the browser, and remembers what it learned. This is the brain.

🚪
The gateway (gateway/)

One front door for every LLM call. It picks a provider, respects rate limits, and writes every call into a cost ledger.

📦
The paper trail (code/state/sessions/)

Every run leaves artifacts on disk: the plan graph, each step's exact prompt and result, and browser screenshots.
